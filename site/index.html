make-markdown-library

Point it at a folder of Word, PDF, PowerPoint, Markdown, HTML, CSV, JSON, and ZIP files. Get back one clean Markdown file an AI can read, plus an index file for rebuilds and automation — and, if you want, one tidy Markdown file per source.

Good for
  • AI — paste one readable file into a chatbot instead of juggling attachments.
  • Storage — keep a whole project's sources together in plain text.
  • Search — find anything across every source at once.
  • Version control — plain text diffs cleanly in Git; binaries don't.
  • Rebuilds — JSON/YAML indexes record hashes, converters, and source offsets.
  • Keeping — Markdown stays readable for decades, with no special software.

SOURCE 01 — use the window

I just want to click something

Run the GUI and a window opens. Choose your folder, pick a converter, press one button. LiteParse can be installed from inside the window.

  1. Install Python (free, one time).
  2. Clone or download the repository.
  3. Open a command window there and run 
    python -m make_markdown_library gui
  4. Choose the folder that holds your files.
  5. Choose MarkItDown, LiteParse, or automatic converter routing.
  6. Tick “also make one file per source” if you want the split version too.
  7. Press Make library. Your Markdown file appears next to your folder.

MarkItDown remains the default converter. LiteParse is optional and can be installed with the GUI button or make-markdown-library setup liteparse.

SOURCE 02 — use a terminal

I'm comfortable with a terminal

Run commands for converter routing, indexes, rebuilds, and library maintenance. 

# make a library: sources/ -> markdown-library.md
make-markdown-library make

# point at any folder, and also write one file per source
make-markdown-library make my-folder -o library.md --individual-files

# add more later, list, remove, check
make-markdown-library make my-folder -o library.md --converter auto --index-format both
make-markdown-library add library.md more.zip
make-markdown-library list library.md
make-markdown-library remove-file library.md 3
make-markdown-library check-file library.md
make-markdown-library doctor
make-markdown-library rebuild markdown-library.index.json

Conversion can use MarkItDown, LiteParse, or automatic routing. In auto/hybrid mode, MarkItDown is tried first and LiteParse can kick in when MarkItDown produces empty text. Duplicate sources are skipped by default; pass --allow-duplicates to keep them.

SOURCE 03 — build with it

I'm a developer

One engine, two faces. The CLI and the window both call the same functions in core. Import it and do the same. 

from make_markdown_library import core

result = core.build_library(
    "sources",
    "markdown-library.md",
    individual_files=True,
    converter_mode="auto",
    index_format="json",
)

print(result.converted_count, result.skipped_count)
for path in result.individual_files:
    print(path)

Every function returns plain data objects, never prints. The same engine supports Markdown-aware ingestion, LiteParse/MarkItDown routing, JSON/YAML indexes, and incremental rebuilds. MIT licensed. Source on GitHub.

outputs

A build writes the combined Markdown library, a human manifest, and a JSON index by default. YAML and one-file-per-source Markdown outputs are optional. The terminal prints the paths and counts so users can see what happened immediately. 

Done. Markdown library created.
  Library:  /path/to/markdown-library.md
  Manifest: /path/to/markdown-library-manifest.md
  JSON index: /path/to/markdown-library.index.json
  Sources included: 12
  Sources skipped:  2
how the file is built

Each source becomes a clearly marked section, so an AI — or you — can always tell where one source ends and the next begins. A short fingerprint spots duplicates, while the index stores full SHA-256 hashes, converter metadata, and source offsets.
