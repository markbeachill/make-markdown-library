make-markdown-library

Turn messy folders, files, and ZIP archives into reproducible AI-readable Markdown libraries with manifests, JSON/YAML indexes, LiteParse-aware PDF routing, and optional one-file-per-source output. 

make-markdown-library make sources -o markdown-library.md --converter auto
Docs → Getting started CLI reference Output reference llms.txt
workflow

One local workflow for AI reading packs

Choose a source folder, select a converter strategy, preserve source boundaries, and write a combined Markdown library plus an index that can be used later for rebuilds and automation.

v3 adds
  • LiteParse options for OCR, images, links, page targeting, DPI, and passwords.
  • Optional PDF complexity routing with lit is-complex.
  • Index schema 1.1 with converter options, fallback details, output counts, and Markdown metadata.
  • CLI output modes: normal summary, verbose, quiet, and JSON.
  • Workflow-first docs and an agent-readable llms.txt.
workflow cards

Common tasks

# Create a library
make-markdown-library make my-folder -o library.md

# Also write one Markdown file per source
make-markdown-library make my-folder -o library.md --individual-files

# Prefer LiteParse for scanned/complex PDFs
make-markdown-library make my-folder -o library.md --converter auto --liteparse-complexity-check

# Tune LiteParse
make-markdown-library make my-folder -o library.md --converter hybrid --liteparse-image-mode placeholder --liteparse-dpi 200

# Preview a rebuild
make-markdown-library rebuild library.index.json --dry-run
converter comparison

Converter strategy

ModeBest forBehaviour
markitdownBroad compatibilityUse MarkItDown only.
liteparsePDF/OCR/layout workUse LiteParse where supported.
autoSafe mixed foldersTry MarkItDown, fallback to LiteParse when output is empty; optional PDF complexity routing.
hybridLayout-heavy projectsPrefer LiteParse for PDFs, MarkItDown for broad format coverage.
