make-markdown-library

Point it at a folder of Word, PDF, PowerPoint, Markdown, HTML, and CSV files. Get back one clean Markdown file an AI can read — and, if you want, one tidy Markdown file per source.

Good for
  • AI — paste one readable file into a chatbot instead of juggling attachments.
  • Storage — keep a whole project's sources together in plain text.
  • Search — find anything across every source at once.
  • Version control — plain text diffs cleanly in Git; binaries don't.
  • Keeping — Markdown stays readable for decades, with no special software.

SOURCE 01 — use the window

I just want to click something

Download one file, run it, and a window opens. Choose your folder, press one button. Nothing else to set up.

  1. Install Python (free, one time).
  2. Download make-markdown-library.py and put it anywhere.
  3. Double-click it — or open a command window there and run 
    python make-markdown-library.py
  4. Choose the folder that holds your files.
  5. Tick “also make one file per source” if you want the split version too.
  6. Press Make library. Your Markdown file appears next to your folder.

The first time you run it, it installs one helper it needs (MarkItDown). That takes a minute and needs internet once; after that it works offline.

SOURCE 02 — use a terminal

I'm comfortable with a terminal

Same one file. Run it with a command instead of opening the window. 

# make a library: sources/ -> markdown-library.md
python make-markdown-library.py make

# point at any folder, and also write one file per source
python make-markdown-library.py make my-folder -o library.md --individual-files

# add more later, list, remove, check
python make-markdown-library.py add library.md more.zip
python make-markdown-library.py list library.md
python make-markdown-library.py remove-file library.md 3
python make-markdown-library.py check-file library.md

Conversion uses MarkItDown, installed automatically on first run. Duplicate sources are skipped by default; pass --allow-duplicates to keep them.

SOURCE 03 — build with it

I'm a developer

One engine, two faces. The CLI and the window both call the same functions in core. Import it and do the same. 

from make_markdown_library import core

result = core.build_library(
    "sources",
    "markdown-library.md",
    individual_files=True,
)

print(result.converted_count, result.skipped_count)
for path in result.individual_files:
    print(path)

Every function returns plain data objects, never prints. The same engine lives in the single make-markdown-library.py and in the installable package (pip install -e . from a clone). MIT licensed. Source on GitHub.

how the file is built

Each source becomes a clearly marked section, so an AI — or you — can always tell where one source ends and the next begins. A short fingerprint lets the tool spot duplicates and trace each source back. 

<!-- markdown-library-file: true -->
======================================================================
SOURCE START
File: report.pdf
Fingerprint: 9963f8ff36b0
======================================================================

The converted text of your source goes here, as plain Markdown.

======================================================================
SOURCE END: report.pdf
======================================================================
