make-markdown-library

Point it at a folder of Word, PDF, PowerPoint, Markdown, HTML, and CSV files. Get back one clean Markdown file an AI can read — and, if you want, one tidy Markdown file per source.

Good for
  • AI — paste one readable file into a chatbot instead of juggling attachments.
  • Storage — keep a whole project's sources together in plain text.
  • Search — find anything across every source at once.
  • Version control — plain text diffs cleanly in Git; binaries don't.
  • Keeping — Markdown stays readable for decades, with no special software.

SOURCE 01 — use the window

I just want to click something

There's a window: choose your folder, press one button. You copy two lines once to open it, then it's all clicking — no commands to learn.

You'll need Python on your computer first. It's a free one-time install.

  1. Open your computer's command window and paste: 
    pip install make-markdown-library
  2. Then paste this to open the window: 
    make-markdown-library gui
  3. Choose the folder that holds your files.
  4. Tick “also make one file per source” if you want the split version too.
  5. Press Make library. Your Markdown file appears next to your folder.

After the first time, you only need the second line — make-markdown-library gui — to open the window again.

SOURCE 02 — use a terminal

I'm comfortable with a terminal

Install once, then make a library from any folder.

# install
pip install make-markdown-library

# make a library: sources/ -> markdown-library.md
make-markdown-library make

# point at any folder, and also write one file per source
make-markdown-library make my-folder -o library.md --individual-files

# add more later, list, remove, check
make-markdown-library add library.md more.zip
make-markdown-library list library.md
make-markdown-library remove-file library.md 3
make-markdown-library check-file library.md

Conversion uses MarkItDown. Duplicate sources are skipped by default; pass --allow-duplicates to keep them.

SOURCE 03 — build with it

I'm a developer

One engine, two faces. The CLI and the window both call the same functions in core. Import it and do the same. 

from make_markdown_library import core

result = core.build_library(
    "sources",
    "markdown-library.md",
    individual_files=True,
)

print(result.converted_count, result.skipped_count)
for path in result.individual_files:
    print(path)

Every function returns plain data objects, never prints. MIT licensed. Source on GitHub.

how the file is built

Each source becomes a clearly marked section, so an AI — or you — can always tell where one source ends and the next begins. A short fingerprint lets the tool spot duplicates and trace each source back. 

<!-- markdown-library-file: true -->
======================================================================
SOURCE START
File: report.pdf
Fingerprint: 9963f8ff36b0
======================================================================

The converted text of your source goes here, as plain Markdown.

======================================================================
SOURCE END: report.pdf
